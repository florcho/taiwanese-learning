# Taiwan Translate (대만 표준중국어 ↔ 한국어)

대만 표준중국어(번체자, zh-TW)와 한국어(ko-KR)를 빠르게 번역하는 스킬.
MCP 서버를 사용하지 않고 직접 번역한다 — 속도가 최우선.

## 언어 자동 감지

입력 텍스트를 보고 방향을 자동으로 정한다. 사용자에게 묻지 않는다.

- 입력이 **한글 위주** → 한국어 → 대만어 (`ko → zh-TW`)
- 입력이 **한자 위주** → 대만어 → 한국어 (`zh-TW → ko`)
- 섞여 있으면 더 많은 쪽, 또는 문장의 주 언어를 기준으로 판단

## 출력 규칙 (방향별로 완전히 다름)

### A. 한국어 → 대만어 (`ko → zh-TW`)

**번역문만 출력한다. 군더더기 금지.**

- 인사말, 설명, "다음은 번역입니다" 같은 머리말 없이 번역 결과 한 덩어리만 출력.
- 반드시 **번체자(繁體字)**. 간체자(简体字) 절대 금지.
- 대만에서 실제로 쓰는 어휘 우선 (본토 중국어 어휘 배제):
  - 軟體(O) / 軟件(X·본토), 影片(O) / 視頻(X·본토), 網路(O) / 網絡(X·본토),
    解析度(O) / 分辨率(X·본토), 滑鼠(O) / 鼠標(X·본토), 品質(O) / 質量(X·본토),
    資訊(O) / 信息(X·본토), 部落格(O) / 博客(X·본토)
- 톤: 인플루언서 DM·비즈니스·일상·구어 맥락. 과하게 딱딱한 문어체로 만들지 말 것.
  공손하되 자연스러운 대만식 구어 톤. 협업 제안·문의 DM이면 您/請/麻煩您 등을
  자연스럽게 섞되 번역투가 되지 않게 한다.
- 한국어 존댓말은 대만식 정중 표현으로 자연스럽게 옮긴다 (직역 금지).

### B. 대만어 → 한국어 (`zh-TW → ko`)

**번역문 + 단어 정리 테이블** 두 부분으로 출력한다.

먼저 자연스러운 한국어 번역문을 출력한다 (DM/비즈니스/구어 톤 유지).
그 아래에 빈 줄을 두고 단어 정리 테이블을 붙인다.

**단어 정리 대상:**
- HSK 4급 이상 단어
- 대만 특유 어휘 / 구어 표현 (HSK 급수와 무관하게 포함)
- **HSK 4급 미만(1~3급)은 제외한다.** 단, 위의 대만 특유/구어 표현은 급수와
  상관없이 포함.
- 같은 단어가 한 번역 안에서 반복되면 한 번만 정리

**테이블 형식 (마크다운):**

```
| 단어 | 병음 | 뜻 |
|------|------|-----|
| 詞彙 | cíhuì | 어휘 |
```

- 병음은 성조 표기 포함 (예: nǐ hǎo). 숫자 성조(ni3 hao3) 금지.
- 대만 특유 어휘/구어는 뜻 칸에 `(대만 구어)` 같은 짧은 라벨을 덧붙인다.
- 정리할 단어가 하나도 없으면 테이블을 생략한다.

## 학습 데이터 누적 → GitHub repo (append)

`--no-log` 플래그가 없는 한, **B 방향(대만어→한국어)** 번역의 원문을
**GitHub 리포 `florcho/taiwanese-learning` 의 `data/corpus.jsonl`** 에 한 줄
append한다. A 방향(한→대만어)은 적재하지 않는다.

⚠️ 옛 방식(`/mnt/user-data/outputs/...` 샌드박스 파일)은 **쓰지 않는다.** 그건
세션 끝나면 사라져서 PWA가 못 읽었다. 이제 **GitHub 커넥터**로 repo에 직접
커밋해야 학습 PWA의 자동 빌드(`build-lessons`)가 집어간다.

스킬은 단어를 쪼개지 않고 **원문 그대로** 적재한다 (단어 정리·注音/拼音·레슨화는
나중에 repo의 build-lessons 에이전트가 한다).

**절차 (GitHub 커넥터 사용):**
1. repo에서 `data/corpus.jsonl` 의 현재 내용을 읽는다.
2. 마지막 줄의 `id`(`cNNNN`)를 보고 다음 번호를 정한다 (없으면 `c0001`).
3. 아래 형식의 **한 줄(JSON)** 을 파일 끝에 덧붙인다 (기존 내용 절대 덮어쓰기 금지):
   ```json
   {"id":"cNNNN","source":"대만어 원문 전체","translation":"한국어 번역문","source_date":"2026-06-10","processed":false,"lessonId":null}
   ```
   - `source`: 입력받은 대만어 원문 전체 (가공 X, 그대로)
   - `translation`: 출력한 한국어 번역문
   - `source_date`: 오늘 날짜 (YYYY-MM-DD)
   - `processed`: 항상 `false` (= 아직 레슨 안 됨 → 자동 빌드 대상)
   - `lessonId`: 항상 `null`
4. `data/corpus.jsonl` 을 commit (메시지 예: `translate-log: cNNNN`).
5. 번역 응답 끝에 한 줄로 알린다:
   `📚 corpus에 적재했어요 (다음 자동 빌드 때 레슨화됨).`

> 화면에 보여주는 단어 테이블과 적재 줄은 별개다. 테이블은 사용자가 바로
> 보라고 만드는 것이고, 적재는 원문만 쌓아 repo가 알아서 레슨화하게 한다.
> 커밋이 실패하면(커넥터 미연결 등) 그 사실을 사용자에게 알린다 — 조용히 넘어가지 말 것.

## 예시

```
한국어 → 대만어:
입력:  안녕하세요! 혹시 협업 제안 드리고 싶어서 연락드렸어요 :)
출력:  您好！想跟您提案合作，所以特地聯繫您 :)
       (그 외 아무 설명 없음)

대만어 → 한국어:
입력:  最近你的限動我都有在看，超有質感的！想問問看有沒有合作的機會？
출력:  요즘 네 스토리 다 보고 있어, 진짜 감각 있더라! 혹시 협업 기회가 있을까 해서.

       | 단어 | 병음 | 뜻 |
       |------|------|-----|
       | 限動 | xiàndòng | 인스타 스토리 (대만 구어, 限時動態 줄임말) |
       | 質感 | zhìgǎn | 감각/퀄리티 (HSK4+) |

       📚 원문을 학습 파일에 추가했어요.
```

## 빠른 처리 원칙

- 짧은 텍스트(한두 문장)는 한 번에 처리하고 끝낸다. 되묻지 않는다.
- 방향이 명확하면 확인 질문 없이 바로 번역한다.
- 긴 텍스트도 분할 협의 없이 그대로 번역한다.
